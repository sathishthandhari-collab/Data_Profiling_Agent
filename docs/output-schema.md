# ProfileReport Schema

This document provides a detailed breakdown of the `ProfileReport` JSON output model generated by **Agent 1: The Data Profiling Agent**. The model is built using **Pydantic v2** to ensure strict typing and data integrity.

## 🧬 Model Overview

The `ProfileReport` is the ultimate output of the agentic profiling workflow. It encapsulates schema metadata, statistical profiles, PII detection, relationship hints, and LLM-driven interpretations into a single, structured object.

## 🧱 Data Structures

### 1. `ProfileReport` (Root)
The primary container for all profiling metadata.

| Field | Type | Description |
| :--- | :--- | :--- |
| `report_id` | `str` | A unique UUID for the profiling run. |
| `source_table` | `str` | Name of the table that was profiled. |
| `source_system` | `str` | The source system the table belongs to (e.g., `core_banking`). |
| `profiled_at` | `datetime` | Timestamp of the profiling execution (UTC). |
| `row_count` | `int` | Total number of rows in the table (before sampling). |
| `columns` | `List[ColumnSchema]` | Basic schema information for each column. |
| `stats` | `List[StatsProfile]` | Statistical distribution and cardinality estimates. |
| `pii_info` | `List[PIIProfile]` | PII classification and confidence scores. |
| `fk_hints` | `List[FKHint]` | Potential Foreign Key relationships found. |
| `interpretation` | `Optional[LLMInterpretation]` | Semantic judgments from the LLM. |
| `schema_version` | `str` | Version of this report model (default: `1.1.0`). |

---

### 2. `ColumnSchema`
Metadata regarding the structure of a specific column.

| Field | Type | Description |
| :--- | :--- | :--- |
| `name` | `str` | Column name. |
| `spark_type` | `str` | The native PySpark data type (e.g., `StringType`, `IntegerType`). |
| `nullable` | `bool` | Whether the column allows null values. |
| `is_pk_candidate` | `bool` | True if the column is a candidate for a Primary Key. |
| `uniqueness_ratio` | `float` | Distinct values / Total rows (0.0 to 1.0). |

---

### 3. `StatsProfile`
Aggregated statistics for a column.

| Field | Type | Description |
| :--- | :--- | :--- |
| `column` | `str` | Name of the column. |
| `null_pct` | `float` | Percentage of null values (0.0 to 100.0). |
| `cardinality_est` | `int` | Estimated number of distinct values (HyperLogLog). |
| `min_val` | `Union[int, float, str]` | Minimum value found in the column. |
| `max_val` | `Union[int, float, str]` | Maximum value found in the column. |
| `mean_val` | `float` | Arithmetic mean (Numeric types only). |
| `stddev_val` | `float` | Standard deviation (Numeric types only). |
| `has_outliers` | `bool` | True if values exceed IQR-based thresholds. |

---

### 4. `PIIProfile`
Sensitive data detection results.

| Field | Type | Description |
| :--- | :--- | :--- |
| `column` | `str` | Name of the column. |
| `is_pii` | `bool` | True if sensitive data was detected. |
| `pii_type` | `Optional[str]` | The category of PII (e.g., `EMAIL`, `SSN`, `NAME`). |
| `confidence` | `float` | Confidence score for the PII detection (0.0 to 1.0). |

---

### 5. `FKHint`
Potential relationship between this table and others in the system.

| Field | Type | Description |
| :--- | :--- | :--- |
| `source_col` | `str` | Column in the current table. |
| `target_table` | `str` | Likely related table in the environment. |
| `target_col` | `str` | Likely related column in the target table. |
| `match_ratio` | `float` | Degree of value containment (0.0 to 1.0). |

---

### 6. `LLMInterpretation`
The result of the `node_interpreter` analyzing raw tool outputs.

| Field | Type | Description |
| :--- | :--- | :--- |
| `suggested_entity_name` | `str` | Semantic business name for the table (e.g., `CustomerAccount`). |
| `bk_candidates` | `List[BKCandidate]` | Ranked list of Business Key candidates. |
| `pii_summary` | `List[str]` | Human-readable summary of sensitive data risks. |
| `dq_flags` | `List[str]` | Data Quality concerns (e.g., "High null rate in PK"). |
| `confidence` | `float` | Overall confidence in the interpretation (0.0 to 1.0). |
